// turbo-all
1. Siempre inicia con una consulta en el MCP de Sanity Antes luego para crear cualquier `implementation_plan.md`, lee el archivo `docs/PRG de Sanity 2026.md` para extraer los enlaces de documentación pertinentes y las mejores prácticas.
2. Cada plan de implementación DEBE incluir una sección de `## Documentation & Reference` al inicio, listando los links relevantes del documento PRG 2026 para la tarea específica.
3. Verifica que el stack propuesto (Next.js App Router, Sanity MCP, Better Auth, Tailwind CSS) coincida exactamente con lo definido en el documento. iniciar MCP Github con el comando: container system start
4. Si la tarea implica cambios en el esquema de Sanity, consulta el documento PRG 2026 para aplicar las "Core Features" recomendadas (TypeGen, Visual Editing, etc.).
5. Realiza el plan extremadamente ultra-concisos. sacrifica la gramática si eso ahorra espacio/tiempo
6. al final de cada plan, crea una lista de tareas para resolver y responder las huvieran.
